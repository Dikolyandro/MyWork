1. В этом задании я использовал паттерн Proxy для онлайн-обучения, чтобы видеоуроки
загружались только при их просмотре.
2. Я создал интерфейс `VideoLecture`, определяющий методы для получения информации и
воспроизведения видеоурока.
3. Реализовал класс `RealVideoLecture`, который имитирует загрузку и воспроизведение видео.
4. В классе `ProxyVideoLecture` добавил ленивую загрузку, чтобы реальный видеоурок
создавался только при первом вызове `play()`.
5. Также создал класс `OnlineCourse`, который управляет несколькими видеоуроками и
позволяет их воспроизводить.
6.В результате, в классе LearningPlatformApp я продемонстрировал, как видеоуроки
загружаются только при воспроизведении, что позволяет экономить ресурсы.

Паттерн Proxy позволяет использовать замещающий объект для контроля доступа к другому
объекту, создавая его только при необходимости. Это экономит ресурсы и может обеспечивать
дополнительные функции, такие как безопасность или логирование.
