1.Создал интерфейс PaymentStrategy, в котором есть метод processPayment.
Этот метод будет использоваться для разных способов оплаты.
2.Реализовал стратегии: три класса для оплаты картой, PayPal и
криптовалютой. Каждый класс содержит свою реализацию метода processPayment, который обрабатывает оплату.
3.Создал класс ShoppingCart, где можно менять стратегию
оплаты через метод setPaymentStrategy, а метод checkout выполняет оплату с выбранной стратегией.
4.В главном классе показал работу системы: сначала корзина
использует оплату картой, потом переключается на PayPal и криптовалюту.

Strategy pattern позволяет менять алгоритмы выполнения задач в зависимости от ситуации,
не изменяя саму программу. В нашем случае это позволяет выбирать и менять способ оплаты
во время выполнения программы.
